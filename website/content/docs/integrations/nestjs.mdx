---
title: NestJS
description: Scope every NestJS request with a module and interceptor — Express or Fastify, any ORM.
---

`tenancyjs-integration-nest` brings tenant scoping to NestJS 11. Register the tenancy module and an
interceptor opens a tenant scope for the lifetime of each request; your controllers and providers run
inside it. It works on both the **Express** and **Fastify** platforms.

> **Works with any ORM.** [Prisma](/docs/adapters/prisma) · [Knex](/docs/adapters/knex) ·
> [TypeORM](/docs/adapters/typeorm) · [Sequelize](/docs/adapters/sequelize) ·
> [Mongoose](/docs/adapters/mongoose).

<Callout type="info">
  NestJS is fully supported, but `tenancy init` doesn't scaffold it yet — this is the manual setup (a
  few lines). See [Installation](/docs/getting-started/installation).
</Callout>

## Install

```bash
npm install tenancyjs-core tenancyjs-integration-nest tenancyjs-adapter-prisma @prisma/client
```

## Wire it up

Provide your `manager` and a resolver to the tenancy module, register the interceptor globally, and pair
it with your ORM's [adapter](/docs/adapters).

```ts title="app.module.ts"
import { Module } from "@nestjs/common";
import { createNestResolverInput } from "tenancyjs-integration-nest";
import { manager } from "./tenancy";

// Register the tenancy module with the manager + a resolver that turns the
// incoming request into a tenant. The interceptor then opens a scope per
// request, so controllers and providers run tenant-scoped.
const resolver = createNestResolverInput({
  manager,
  resolve: (req) => ({ id: req.headers["x-tenant"] ?? "" }),
});

@Module({
  // ...register the tenancy module + interceptor with `resolver`
})
export class AppModule {}
```

## How it behaves

- Each request resolves its tenant and runs inside the scope; ORM queries through the adapter are scoped.
- A resolution failure fails closed with the correct HTTP status **before** your controller runs.
