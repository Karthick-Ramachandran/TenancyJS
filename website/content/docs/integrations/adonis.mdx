title: AdonisJS
description: Wire Lucid tenancy into the AdonisJS request lifecycle via a provider and middleware.
---

`tenancyjs-integration-adonis` connects [Lucid tenancy](/docs/adapters/lucid) to the AdonisJS 7 request
lifecycle. Because Lucid supports **all three isolation strategies**, AdonisJS gets the full range —
row-level, schema-per-tenant, and database-per-tenant — out of the box.

<Callout type="info">
  `tenancy init` scaffolds AdonisJS + Lucid end-to-end, so the fastest path is
  [`npx tenancy init`](/docs/getting-started/installation). This page covers the manual wiring.
</Callout>

## Install

```bash
npm install tenancyjs-integration-adonis
```

## Configure

Define your tenancy config with `defineAdonisTenancyConfig`, providing a resolver that turns the request
context into a tenant:

```ts title="config/tenancy.ts"
import { defineAdonisTenancyConfig } from "tenancyjs-integration-adonis";

export default defineAdonisTenancyConfig({
  resolver: (ctx) => ({ id: ctx.subdomains.tenant ?? "" }),
});
```

Register the provider, then add the tenancy middleware to your router (the `init` scaffold does both for
you). Each request resolves its tenant and runs scoped, so your Lucid models are automatically isolated
— no per-query tenant filters.

## How it behaves

- The middleware opens a tenant scope for the request; Lucid queries run scoped through the adapter.
- Resolution failures fail closed with the right status before your controller runs.
- Central-scope work (cross-tenant admin) is opened explicitly, never by accident.
