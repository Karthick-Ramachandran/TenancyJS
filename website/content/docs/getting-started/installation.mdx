## Requirements

- **Node.js 24+** — the toolkit and CLI use modern Node features, and the CLI loads your TypeScript
  config via Node's native type-stripping (no transpiler needed).
- TypeScript is recommended but not required.

## Recommended: scaffold with the CLI

The fastest way to get set up. `tenancy init` detects your framework and ORM, then writes a working
`tenancy.config.ts`, your tenant registration, and the request middleware — correctly wired for your
stack. No copy-pasting boilerplate.

```bash
npx tenancy init
```

### Which stacks does `init` scaffold?

| Framework | ORM | `tenancy init` | Guide |
| --- | --- | :---: | --- |
| Express | Prisma | ✅ scaffolded | [Express](/docs/integrations/express) |
| AdonisJS | Lucid | ✅ scaffolded | [AdonisJS](/docs/integrations/adonis) |
| Next.js | Prisma | ✅ scaffolded | [Next.js](/docs/integrations/nextjs) |
| **NestJS** | any adapter | manual (a few lines) | [NestJS](/docs/integrations/nestjs) |

<Callout type="info">
  **On NestJS, or a different ORM (Knex, TypeORM, Sequelize, Mongoose)?** It's fully supported — the CLI
  just doesn't scaffold it *yet*. Use the [Manual setup](#manual-setup) below together with your
  [integration guide](/docs/integrations) (e.g. [NestJS](/docs/integrations/nestjs)) and
  [adapter guide](/docs/adapters). It's only a few lines of wiring.
</Callout>

`init` writes the wiring; add the packages it uses for your stack (for example, Express + Prisma):

```bash
npm install tenancyjs-core tenancyjs-adapter-prisma tenancyjs-integration-express
```

Then jump to the [Quickstart](/docs/getting-started/quickstart) to see it working.

### The CLI does more than scaffold

`init` is just the start. The same CLI lists, creates, migrates, provisions, and runs scripts against
your live tenants. See the full **[CLI reference →](/docs/cli)**.

## Manual setup

TenancyJS is a set of small, composable `tenancyjs-*` packages. Install the core, then add the adapter
for your ORM and the integration for your framework.

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">```bash npm install tenancyjs-core ```</Tab>
  <Tab value="pnpm">```bash pnpm add tenancyjs-core ```</Tab>
  <Tab value="yarn">```bash yarn add tenancyjs-core ```</Tab>
  <Tab value="bun">```bash bun add tenancyjs-core ```</Tab>
</Tabs>

| Package | Role |
| --- | --- |
| `tenancyjs-core` | Framework-neutral tenant context, lifecycle, and the runtime contract |
| `tenancyjs-identifiers` | Resolve a tenant from a request (subdomain, header, path…) |
| `tenancyjs-adapter-prisma` · `-knex` · `-lucid` · `-typeorm` · `-sequelize` · `-mongoose` | Enforce isolation inside your ORM ([guides](/docs/adapters)) |
| `tenancyjs-integration-express` · `-next` · `-adonis` · `-nest` | Bind context to the request lifecycle ([guides](/docs/integrations)) |
| `tenancyjs-cli` | The operational CLI (`tenancy …`) |
| `tenancyjs-testing` | Fixtures and conformance helpers for your own tests |

<Callout type="warn">
  During the beta the packages are published under the `beta` dist-tag. A plain `npm install
  tenancyjs-core` resolves the latest published beta; pin an exact version for reproducibility.
</Callout>
